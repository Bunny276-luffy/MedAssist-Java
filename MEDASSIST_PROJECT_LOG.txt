--------------------------------------------------------------------------------
  HOW TO RUN
--------------------------------------------------------------------------------

Step 1: Set JAVA_HOME (only needed in current PowerShell session if Java was
        just installed):

        $env:JAVA_HOME = "C:\Program Files\Eclipse Adoptium\jdk-17.0.18.8-hotspot"
        $env:Path = "$env:JAVA_HOME\bin;" + $env:Path

Step 2: Navigate to project:

        cd F:\projects\java\medassist

Step 3: Compile:

        mvn compile

Step 4: Run:

        mvn exec:java

Note: Run compile and exec:java as SEPARATE lines in PowerShell.
      '&&' is NOT supported in older PowerShell versions.

--------------------------------------------------------------------------------
  LAYER 2 — SERVICE (com.medassist.service)
--------------------------------------------------------------------------------

Notifiable.java (Interface)
  Methods  : sendNotification(String), escalate(Patient, Medication)

BaseReminder.java (Abstract Class)
  Fields   : ScheduledExecutorService scheduler
  Methods  : scheduleReminder(), cancelReminder(), abstract fireAlert()
  Purpose  : Base class for all reminder types (OOP inheritance)

ReminderService.java (extends BaseReminder, implements Notifiable)
  Purpose  : Schedules medication reminders using ScheduledExecutorService
  Features :
    - Calculates delay in seconds until each medication's scheduledTime
    - Checks ActivityMonitor.isUserActive() before firing alert
    - If patient is inactive (sleep hours or random probability), delays
      reminder by 2 minutes
    - Maintains List<Notifiable> for polymorphic notification dispatch
    - Fires both voice (VoiceEngine) and email (EscalationService) alerts

OCRService.java
  Purpose  : Scans prescription images and extracts medication details
  Features :
    - Uses Tess4J (loaded via Java reflection to avoid crash if missing)
    - Falls back to simulation mode if Tesseract native libs not found
    - Delegates parsed text to RegexParser

EscalationService.java (implements Notifiable)
  Purpose  : Monitors missed doses and emails the caregiver
  Features :
    - Background ScheduledExecutorService polls every 5 minutes
    - Sends HTML-formatted alert email via JavaMail / Gmail SMTP
    - Deduplicates alerts (does not send duplicate emails)
    - Persists escalation log to data/escalation_log.txt
    - Provides triggerTestAlert() for UI testing

ActivityMonitor.java
  Purpose  : Simulates patient activity detection
  Logic    :
    - Returns inactive during 22:00–06:00 (sleep hours)
    - During waking hours: 70% probability of being active
    - Used by ReminderService to delay reminders for inactive patients

  LAYER 3 — VOICE (com.medassist.voice)
--------------------------------------------------------------------------------

SpeechCapable.java (Interface)
  Methods  : speak(String, Language), listen(), parseCommand(String)

LanguagePack.java
  Purpose  : Multilingual string repository
  Keys     : greeting, reminder, taken_confirm, missed_alert, escalation_sent
  Languages: English, Telugu, Hindi

VoiceEngine.java (implements SpeechCapable)
  Version 1 Features:
    - speak() — prints localised text to console with language tag
    - listen() — cycles through demo commands
    - parseCommand() — maps keywords to CMD_* constants

  Version 2 Enhancements (natural language processing):
    - setInput(String) — UI injects typed/transcribed text
    - listen() — returns injected input or falls back to demo commands
    - extractMedicationDetails(String input) — NLP extraction:
        * DRUG_NAME_PATTERN  — extracts drug name after "add"/"mark"/"schedule"
        * DOSAGE_PATTERN     — extracts "100mg", "2 tablets", "500ml", etc.
        * TIME_PATTERN       — extracts "at 9am", "at 21:00", "at 9:30pm"
        * FREQUENCY_PATTERN  — extracts "once daily", "twice daily", etc.
        * MISSED_DRUG_PATTERN — extracts drug from "I missed my X dose"
        * TAKEN_DRUG_PATTERN  — extracts drug from "I have taken X"
    - Returns Map<String, String> with keys: drugName, dosage, time, frequency
    - parseTime() — converts hour/min/ampm to 24-hour "HH:mm" format
    - cleanDrugName() — removes noise words (dose, medication, tablet) from name

WindowsSpeechRecognizer.java (NEW)
  Purpose  : Real microphone speech recognition for Windows
  How it works:
    - Spawns a PowerShell process with an inline System.Speech script
    - Uses System.Speech.Recognition.SpeechRecognitionEngine (built into .NET)
    - Loads DictationGrammar for free-form speech
    - Settings: InitialSilenceTimeout=6s, EndSilenceTimeout=2s, BabbleTimeout=0
    - Records up to 10 seconds of audio from default microphone
    - Returns top recognition result + alternate hypotheses (newline-separated)
    - Non-blocking: uses ExecutorService + Consumer callbacks (onResult, onError)
    - Requires: Windows + .NET Framework 3.5+ (no extra downloads needed)

VoicePanel.java (Major upgrades across 2 iterations)
  Version 1:
    - Language selector chips (English / Telugu / Hindi)
    - "🎤 Speak" button cycling demo commands
    - Greet button
    - Monospace output terminal area

  Version 2:
    - Text input field with placeholder ("Type a command or press 🎙️ to speak")
    - Example chips (clickable): "add Aspirin 100mg at 9am",
      "mark Metformin as taken", "I missed my Amlodipine dose",
      "schedule Atorvastatin at 9pm", "scan prescription"
    - Process ▶ button (purple) — also triggered by pressing Enter
    - 🎙️ Record button (gray, turns red when recording)

  Voice Recording Flow:
    1. Click 🎙️ — button turns ⏹, border turns red, field shows "🔴 Listening…"
    2. Timer counts seconds elapsed
    3. Windows Speech API captures audio (via PowerShell)
    4. After recognition: Review & Confirm dialog appears
    5. Dialog shows:
       - "I heard:" label
       - Editable text field with best recognition result
       - Clickable alternate chips (other recognition hypotheses)
       - "Correct any mistakes, then press Process." hint
    6. OK → auto-processes the corrected command
    7. Cancel → puts text in main field for manual editing

  Command Execution (real actions):
    CMD_ADD:
      - Extracts drug name, dosage, time, frequency from NLP
      - Creates new Medication object and adds to patient list
      - Saves to patient.dat immediately
      - Returns: "✅ Added Aspirin (100mg) scheduled at 09:00 — once daily."

    CMD_TAKEN:
      - Extracts drug name from NLP
      - If name found: finds matching medication (fuzzy case-insensitive match)
      - Marks it as TAKEN, saves to patient.dat
      - If no name: marks first PENDING medication as taken
      - Response in selected language (English/Telugu/Hindi)

    CMD_MISSED:
      - Extracts drug name from NLP
      - Finds matching medication and marks it MISSED
      - Saves to patient.dat
      - Response in selected language

    CMD_SCHEDULE:
      - Extracts drug name and new time
      - Finds matching medication and updates scheduledTime
      - Saves to patient.dat

    CMD_SCAN:
      - Responds: "Navigate to the Scan Rx tab"

    CMD_UNKNOWN:
      - "I couldn't understand that command. Try one of the examples above."

  Output terminal:
    - Dark background, green monospace text
    - Shows: Input / Command / Parsed details / Result for every interaction
    - Auto-scrolls to latest entry

--------------------------------------------------------------------------------
  SETUP ISSUES ENCOUNTERED & SOLUTIONS
--------------------------------------------------------------------------------

Issue 1: 'mvn' is not recognized
  Cause  : Maven not installed
  Fix    : Downloaded Maven 3.9.6, added to PATH
           OR used Maven Wrapper (mvnw.cmd) added to project

Issue 2: mvnw.cmd failed — PowerShell multi-line quoting broke download
  Fix    : Manually downloaded maven-wrapper.jar using Invoke-WebRequest

Issue 3: JAVA_HOME not found
  Cause  : Java installed but PATH not updated in current session
  Fix    : winget install EclipseAdoptium.Temurin.17.JDK
           Then refreshed PATH:
           $env:JAVA_HOME = "C:\Program Files\Eclipse Adoptium\jdk-17.0.18.8-hotspot"
           $env:Path = "$env:JAVA_HOME\bin;" + $env:Path

Issue 4: '&&' not valid in PowerShell
  Fix    : Run mvn compile and mvn exec:java as separate lines

Issue 5: Voice said "paracetamol" but recognised as "The rest of the"
  Cause  : Windows Speech Recognition accuracy for medical terms / accents
  Fix 1  : Improved PowerShell script with:
             InitialSilenceTimeout = 6 seconds
             EndSilenceTimeout = 2 seconds
             BabbleTimeout = 0
             Returns up to 3 alternate recognition hypotheses
  Fix 2  : Added Review & Confirm dialog showing what was heard,
             with editable field + alternate chips before processing
